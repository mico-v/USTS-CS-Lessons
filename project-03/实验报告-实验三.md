## 二、实验目的与任务

本实验围绕继承、构造方法链、方法重写（Override）、方法重载（Overload）、this 与 super 关键字展开，目标是：

- 掌握多层继承中构造方法执行顺序。
- 掌握重写与重载的语法和语义差异。
- 能使用 super 调用父类构造方法与被重写方法。
- 理解 final 方法“可继承不可重写”的限制。

## 三、实验内容与代码实现

### 1. Ex1_ConstructorChain.java

实现点：

- 为 `Vehicle` 增加有参构造，初始化 `brand` 并打印提示。
- 让 `Car` 继承 `Vehicle`，通过 `super(brand)` 完成父类初始化。
- 让 `ElectricCar` 继承 `Car`，通过 `super(brand, seats)` 继续构造方法链。

关键思路：

- 子类构造方法第一行必须是 `super(...)` 或 `this(...)`。
- 构造顺序固定为：父类 -> 子类。

### 2. Ex2_OverrideAndThis.java

实现点：

- 在 `Circle` 中重写 `getInfo()`、`getArea()`、`describe()`。
- `getArea()` 使用 `Math.PI * r * r` 并保留两位小数。
- 在 `showBoth()` 中分别调用 `super.describe()` 与 `this.describe()` 对比效果。

关键思路：

- `super.describe()` 强制调用父类实现。
- `this.describe()` 调用当前对象的实际类型实现（此处为 `Circle`）。

### 3. Ex3_OverrideVsOverload.java

实现点：

- 重写 `add(int, int)`，增加科学计算器标记输出。
- 重载 `add(double, double)` 与 `add(int, int, int)`。
- 新增子类特有方法 `square(double)`。
- 在 `compareAdd()` 中对比 `super.add(...)` 与 `this.add(...)`。

关键思路：

- 重写：方法名与参数列表都相同，发生在父子类之间。
- 重载：方法名相同但参数不同，发生在同一类中。

## 五、问题与解决

1. 构造方法链报错：子类若未显式调用可用的父类构造方法，会出现编译错误。
解决：在子类构造方法第一行使用 `super(...)`，参数与父类构造签名保持一致。

2. 重写与重载容易混淆：仅方法名相同并不一定是重写。
解决：确认“重写必须参数列表完全一致，重载必须参数列表不同”。

3. `final` 方法误重写：取消注释后会触发编译错误。
解决：保留注释，仅调用父类提供的 `final` 方法。

## 六、实验总结与体会

通过本次实验，我将继承链、构造方法链、方法重写和方法重载放在同一组案例中进行对比，理解更清晰：

- 构造方法链体现了“先父后子”的初始化规则。
- super 与 this 分别代表“父类实现”和“当前对象实现”。
- Override 与 Overload 的判断标准应以参数列表为核心。

本实验完成后，能够更准确地设计父子类职责，并在调试中快速定位“调用的是父类版本还是子类版本”。
