# Java 类与对象综合实验

## 一、实验目的

本实验旨在帮助学生掌握 Java 语言中类与对象的核心内容，通过实操训练强化代码编写与调试能力。通过本实验，学生将：

* 理解类的定义方式，掌握属性声明、构造方法重载以及封装（private 修饰符配合 getter/setter）的基本用法。
* 理解 static 关键字的含义，能够区分类变量与实例变量、静态方法与实例方法的作用范围与访问规则。
* 通过实验观察，深入理解 Java 中基本类型的值传递与引用类型的地址传递机制，明确方法调用时参数传递的实际行为。
* 掌握对象引用赋值的含义，能够区分"复制引用"与"复制对象"，理解 == 运算符对引用类型的比较规则。
* 了解静态嵌套类与内部类（非静态成员类）的定义方式、创建语法及其对外部类成员的访问权限差异。
* 通过综合性题目，将上述知识点融会贯通，培养面向对象的程序设计思维和独立调试排错的能力。

## 实验原理

* 类与对象原理： 类是对一类事物的抽象描述，定义了该类事物所具有的属性（成员变量）和行为（成员方法）。对象是类的具体实例，通过 new 关键字调用构造方法在堆内存中分配空间并完成初始化。一个类可以创建多个对象，各对象拥有独立的实例变量副本。
* 封装原理： 封装是面向对象的基本特征之一。通过将属性声明为 private，禁止外部直接访问，再提供 public 的 getter 和 setter 方法作为受控的访问接口，可以在 setter 中加入合法性校验逻辑，从而保证对象内部数据的安全性和一致性。
* 构造方法原理： 构造方法是与类同名且无返回值类型的特殊方法，在 new 创建对象时自动调用。Java 支持构造方法重载，即同一个类中可以定义多个参数列表不同的构造方法，以满足不同的初始化需求。当类中未显式定义任何构造方法时，编译器会自动提供一个无参的默认构造方法；一旦显式定义了构造方法，默认构造方法将不再自动生成。
* static 关键字原理： 被 static 修饰的变量和方法属于类本身而非某个具体对象。静态变量在内存中只有一份，被该类的所有实例共享；静态方法可以通过类名直接调用，但其内部不能直接访问非静态的实例成员，因为静态方法执行时不依赖于任何对象实例。
* 参数传递原理： Java 中方法的参数传递统一采用值传递机制。对于基本数据类型，传递的是值的副本，方法内对形参的修改不影响实参；对于引用类型，传递的是引用（即对象地址）的副本，因此方法内通过该引用修改对象属性会影响原对象，但若在方法内将形参指向新对象，则不会改变实参所指向的对象。
* 内部类原理： Java 允许在一个类的内部定义另一个类。静态嵌套类（static nested class）用 static 修饰，不持有外部类实例的引用，可以独立创建，但不能直接访问外部类的实例成员。非静态内部类（inner class）隐含持有一个外部类实例的引用，因此可以直接访问外部类的所有成员（包括私有成员），但其创建必须依赖一个外部类的实例对象。
